\## Overview



Enterprise Zero Trust Architecture is a production-inspired cybersecurity project that demonstrates how modern Zero Trust principles can be implemented within a hybrid enterprise environment.



The solution combines traditional enterprise networking, on-premises identity services, cloud identity integration, secure application access, centralized monitoring, and Infrastructure as Code (IaC) into a single, documented architecture.



The objective is not only to deploy these technologies, but also to demonstrate how they work together as part of a complete enterprise security strategy.



\---



\# Project Vision



Modern organizations can no longer rely solely on perimeter-based security.



This project demonstrates an architecture where every user, device, application, and connection is continuously verified before access is granted.



The environment follows the Zero Trust principle:



> \*\*Never Trust. Always Verify.\*\*



Authentication, authorization, and continuous validation are enforced regardless of whether users are located inside or outside the corporate network.



\---



\# Objectives



The primary objectives of this project are:



\- Design a production-inspired enterprise infrastructure

\- Implement Zero Trust Network Access (ZTNA)

\- Integrate Microsoft Active Directory with Microsoft Entra ID

\- Secure internal applications using Zscaler Private Access (ZPA)

\- Deploy centralized monitoring using Wazuh SIEM

\- Automate infrastructure deployment with Terraform

\- Produce enterprise-grade technical documentation

\- Demonstrate security best practices and modern identity-based access control
